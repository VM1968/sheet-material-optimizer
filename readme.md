# 📐 Оптимизатор раскроя листового материала

Веб-приложение для оптимального раскроя листового материала (фанеры, ДСП, металла и т.д.) с минимизацией отходов и поддержкой множества стандартных размеров.

## 🎯 Возможности

- ✅ **Расчет оптимальных схем раскроя** - алгоритм расчета лучших вариантов расположения деталей
- ✅ **Визуализация на холсте** - наглядное отображение расположения деталей на листе
- ✅ **Расчет стоимости** - автоматический расчет необходимого количества листов и стоимости материала
- ✅ **Анализ отходов** - статистика по использованию материала и объему отходов
- ✅ **Ротация деталей** - возможность фиксации ориентации или свободного выбора оптимальной ориентации
- ✅ **Множество листов** - поддержка работы с листами разных размеров и стоимости
- ✅ **Стандартные размеры** - библиотека популярных размеров листов (фанера, ДСП, МДФ, металл, пластик и т.д.)
- ✅ **Печать** - вывод готовых схем раскроя на печать в удобном формате
- ✅ **Отзывчивый дизайн** - работает на мобильных устройствах, планшетах и ПК

## 🛠️ Технологический стек

- **Frontend**: HTML5, CSS3, Bootstrap 5, JavaScript (vanilla)
- **Backend**: PHP 7.4+
- **Визуализация**: Canvas API
- **Алгоритм**: Улучшенный алгоритм размещения с поддержкой ротации

## 📋 Требования

- PHP 7.4 или выше
- Веб-сервер (Apache, Nginx и т.д.)
- Современный браузер с поддержкой Canvas API

## 🚀 Установка

1. Скачайте или клонируйте репозиторий:
```bash
git clone https://github.com/VM1968/sheet-material-optimizer.git
cd sheet-material-optimizer
```

2. Разместите файлы на веб-сервере

3. Откройте приложение в браузере

## 📖 Инструкция по использованию

### 1. Управление листами

- **Добавить лист**: Нажмите кнопку "+ Добавить лист"
- **Стандартные размеры**: Нажмите "📦 Стандартные размеры" для быстрого выбора популярных размеров
- **Редактировать лист**: Измените длину, ширину, стоимость и припуск при резке
- **Удалить лист**: Нажмите ✕ на карточке листа

### 2. Добавление деталей

- Введите размеры детали (длина и ширина в мм)
- Укажите количество деталей
- Выберите режим поворота:
  - **Свободно** - приложение сможет автоматически выбрать оптимальную ориентацию
  - **Зафиксировано** - деталь не будет поворачиваться
- Нажмите "Добавить деталь"

### 3. Оптимизация раскроя

1. Нажмите кнопку "🚀 Оптимизировать раскрой"
2. Приложение рассчитает оптимальные схемы раскроя
3. Вы увидите:
   - Количество необходимых листов
   - Полную стоимость материала
   - Процент использования материала
   - Объем отходов

### 4. Визуализация схем

- Каждая схема раскроя отображается на холсте Canvas
- Разные цвета обозначают разные детали
- На каждой детали указаны её размеры

### 5. Печать

1. После оптимизации нажмите кнопку "🖨️ Печать"
2. В новом окне откроется готовый документ для печати
3. Документ содержит:
   - Заголовок и дату создания
   - Статистику по использованию материала
   - Все схемы раскроя с размерами листов
4. Используйте стандартную функцию печати браузера (Ctrl+P или ⌘+P)

## 🔧 Технические детали

### Алгоритм оптимизации

1. **Сортировка деталей** - детали сортируются по площади (большие первыми)
2. **Размещение** - для каждой детали находится оптимальная позиция на листе
3. **Ротация** - если включена, алгоритм пытается найти лучшую ориентацию
4. **Генерация схем** - когда на текущем листе нет места, начинается новый лист

### API

**POST /api.php**

Параметры:
- `action` - действие (например, 'optimize')
- `sheet_length` - длина листа в мм
- `sheet_width` - ширина листа в мм
- `material_cost` - стоимость листа в рублях
- `margin` - припуск при резке в мм
- `details` - JSON-массив деталей с полями: length, width, quantity, rotation

Ответ:
```json
{
  "success": true,
  "sheets_needed": 2,
  "total_cost": 2000,
  "material_used": 85.5,
  "waste": 14.5,
  "patterns": [...]
}
```

## 📝 Стандартные размеры листов

- Фанера: 2500×1250 мм
- ДСП: 2500×1250 мм
- МДФ: 2750×1830 мм
- Листовой металл: 2000×1000 мм
- Пластик: 2400×1200 мм
- Стекло: 3000×2000 мм

## 🐛 Известные ограничения

- Алгоритм использует эвристический подход, поэтому может найти решение, но не гарантирует абсолютный оптимум
- Для очень большого количества деталей расчет может занять некоторое время
- Максимальная точность зависит от размера сетки позиций
